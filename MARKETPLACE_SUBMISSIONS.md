## 1. Cline MCP Marketplace

**Submit via:** GitHub Issue at https://github.com/cline/mcp-marketplace/issues/new

**Required:**
- GitHub Repo URL: `https://github.com/VibeCodingInc/vibe-mcp`
- Logo (400x400 PNG): `https://raw.githubusercontent.com/VibeCodingInc/vibe-mcp/main/assets/logo.png`

**Submission text:**

```markdown
## MCP Server Submission: /vibe

**Repository:** https://github.com/VibeCodingInc/vibe-mcp

**Logo:** https://raw.githubusercontent.com/VibeCodingInc/vibe-mcp/main/assets/logo.png

### Why add /vibe?

/vibe is the social layer for AI-assisted coding. It lets developers:
- See who's online and what they're building
- Send DMs without leaving the editor
- Share context and ship together
- Discover other builders working on similar problems

**Community adoption:**
- Published on npm with 22 versions (slashvibe-mcp)
- Listed on MCP Registry, Smithery, Glama
- First MCP server with live presence widget (MCP Apps)

**Developer credibility:**
- Open source (MIT)
- 531 tests passing
- Active development (43 tools)

**Project maturity:**
- Production-ready API (slashvibe.dev)
- Universal installer: `npx slashvibe-mcp install`
- Works with Claude Code, Cursor, VS Code, Windsurf, Cline, Continue.dev, JetBrains

**Security:**
- GitHub OAuth for identity
- No secrets stored locally
- Privacy-first design

### Tested with Cline?

Yes. Install via:
```bash
npx slashvibe-mcp install
```

Or add manually to MCP settings.
```
